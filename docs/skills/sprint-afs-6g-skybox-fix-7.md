## SCOPE

The `deep_space_01.png` texture from spacespheremaps.com is fundamentally a sparse star field — it has very little nebula content to brightness-multiply. Live verification after fix-6 (brightness=4.0) showed avgSum 43.8 with no purple/red tints visible — math proved the texture itself is the bottleneck, not any pipeline issue.

User generated a custom AI texture matching the desired vision: vibrant distant nebula clouds (purple, blue, red) on deep space void with stars baked in. Tested at 1774×887 native resolution, 2.000 ratio (perfect equirectangular), 353 KB file size.

This sprint swaps battle scene to use the new texture and resets brightness to 1.0 since the new texture is naturally saturated.

    const off = document.createElement('canvas');

    const ctx = off.getContext('2d');
    ctx.drawImage(canvas, 0, 0);
    const points = [
      [639, 80], [250, 100], [1028, 100], [200, 270], [1078, 270],
      [350, 450], [928, 450], [639, 500], [639, 600]
    ];
    let sums = []; let tints = {purple:0, blue:0, red:0, neutral:0};
    for (const [x,y] of points) {
      const d = ctx.getImageData(x, y, 1, 1).data;
      const sum = d[0]+d[1]+d[2];
      sums.push(sum);
      let tint = 'neutral';
      if (d[2] > d[0]+5 && d[2] > d[1]+5) tint = d[0] > d[1]+3 ? 'purple' : 'blue';
      else if (d[0] > d[1]+5 && d[0] > d[2]+5) tint = 'red';
      tints[tint]++;
    }
    resolve({avgSum: (sums.reduce((a,b)=>a+b,0)/sums.length).toFixed(1), maxSum: Math.max(...sums), over50: sums.filter(s=>s>50).length, over100: sums.filter(s=>s>100).length, tints});
  });
})

## RISKS

1. **Resolution may be too low for sphere mapping.** Texture is 1774×887, deployed on sphere radius 1500. Close-up might show pixelation. Mitigation: live verify; if pixelated, upscale via Topaz/ESRGAN/real-esrgan in fix-8 (low priority).

2. **PNG-to-sphere wrapping seam.** AI-generated images don't always have seamless horizontal edges. If a vertical seam is visible at one azimuth angle, mitigation: add seam-blending in image editor (5-min Photoshop fix) or accept it (camera azimuth is limited ±20° in OrbitControls — seam may be permanently out of view).

3. **Texture pole compression.** Top/bottom 10% of equirectangular textures get squeezed at the sphere poles. AI-generated images may not have this baked in. Live verify will reveal — likely invisible because OrbitControls polar limits keep poles out of view.

4. **License attribution gap.** New texture is "internal voidexa asset" but if any AI training-data ToS applies (e.g. ChatGPT image gen has commercial use restrictions on certain plans), document the source tool in README. Jix to confirm.
